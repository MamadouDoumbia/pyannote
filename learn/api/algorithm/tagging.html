Label tagging¶

Label tagging algorithms are defined in module pyannote.algorithm.tagging.label.

They can be used to propagate labels from one Annotation (called source) to another Annotation (called target).

They act as translation algorithms where each target label is either given a unique source label translation or left unchanged.

class pyannote.algorithm.tagging.label.LabelTagger(mapper)[source]¶

Generic label tagging.

Label tagging algorithms are made of two steps:
  • first, a label mapping algorithm is applied to find the optimal mapping between target labels and source labels.
  • then, any target label with a corresponding source label is translated using this optimal mapping.
Parameters :

mapper : any BaseMapper subclass

Mapping algorithm used to find the optimal mapping between source and target labels.

Segment tagging¶

Module pyannote.algorithm.tagging.segment provides segment-wise tagging algorithms.

class pyannote.algorithm.tagging.segment.ConservativeDirectTagger[source]¶

Conservative direct segment tagger

Only supports annotation tagging.

class pyannote.algorithm.tagging.segment.ArgMaxDirectTagger[source]¶

ArgMax direct segment tagger

It supports both timeline and annotation tagging.

Timeline tagging. Each segment in target timeline is tagged with the N intersecting labels with greatest co-occurrence duration:

  • N is set to 1 in case source annotation is single-track.
  • In case of a multi-track source, N is set to the the maximum number of simultaneous tracks in intersecting source segments.
